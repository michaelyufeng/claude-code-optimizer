# 代码审查

请对以下内容进行代码审查：$ARGUMENTS

## 审查维度

### 🔴 安全性（优先级最高）
- SQL 注入风险
- XSS 漏洞
- 认证/授权问题
- 敏感信息泄露
- 输入验证缺失

### 🟡 Bug 风险
- 空值/边界检查
- 错误处理
- 逻辑漏洞
- 竞态条件

### 🔵 代码质量
- 命名规范
- 代码重复
- 函数过长
- 复杂度过高

### ⚡ 性能
- N+1 查询
- 不必要的循环
- 内存泄漏风险
- 缓存使用

## 输出格式

```markdown
## 审查结果

### 🔴 严重问题 (必须修复)
**文件**: `src/xxx.ts:45`
**问题**: [描述]
**修复建议**: [具体代码]

### 🟡 中等问题 (建议修复)
...

### 🔵 轻微问题 (可选优化)
...

### ✅ 做得好的地方
...
```

## 审查步骤

1. 先读取相关文件
2. 按照上述维度逐一检查
3. 输出结构化的审查结果
4. 对于严重问题，提供具体修复代码
