# 任务规划

请为以下任务制定详细计划：$ARGUMENTS

## 步骤

### 1. 理解需求
- 仔细阅读任务描述
- 识别关键目标和约束
- 列出不清楚的地方

### 2. 调研现状
- 搜索相关代码文件
- 了解现有实现方式
- 检查是否有类似功能可参考

### 3. 制定方案
请输出：

```markdown
## 任务目标
[简述目标]

## 实现方案
[分步骤描述]

## 涉及文件
- [ ] file1.ts - 修改原因
- [ ] file2.ts - 修改原因

## 风险评估
- 风险1：[描述] → 应对：[措施]

## 工作量
- 预计修改 X 个文件
- 复杂度：低/中/高
```

### 4. 确认
输出计划后，等待用户确认后再开始实现。

**重要：不要直接开始写代码，先输出计划！**
