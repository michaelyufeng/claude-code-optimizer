# 任务检查点

记录当前任务进展，以便上下文切换或新对话时恢复。

## 当前状态

请输出以下信息到 `CHECKPOINT.md`：

```markdown
# 任务检查点 - [时间戳]

## 📋 任务概述
$ARGUMENTS

## ✅ 已完成
- [列出已完成的步骤]

## 🔄 进行中
- [当前正在做什么]
- 当前文件：
- 当前行号：

## 📝 待完成
- [列出剩余步骤]

## 🧠 上下文
### 关键决策
- [记录重要的设计决策]

### 遇到的问题
- [问题描述] → [解决方案/待解决]

### 相关文件
- `file1.ts` - [为什么重要]
- `file2.ts` - [为什么重要]

## 🔗 恢复命令
下次继续时，运行：
\`\`\`bash
# 查看检查点
cat CHECKPOINT.md

# 查看相关文件
cat [关键文件路径]
\`\`\`

## ⚠️ 注意事项
- [任何需要记住的事项]
```

## 使用场景

### 上下文将满时
当感觉上下文快用完了，运行此命令保存进度。

### 需要切换任务时
保存当前进度，方便后续恢复。

### 长任务分段时
定期保存检查点，防止进度丢失。

## 恢复流程

在新对话中：
1. 读取 `CHECKPOINT.md`
2. 了解任务背景
3. 从"进行中"继续
