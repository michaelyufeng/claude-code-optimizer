# 研究阶段 (Phase 1/9)

## 前置检查

首先读取 `.claude/PROJECT_STATE.json` 确认：
1. 项目类型是否为 `new`
2. 当前阶段是否为 `research`

如果不是，提示用户当前状态并拒绝执行。

## 研究任务

你必须引导用户完成以下 6 类研究。每完成一类，更新 `PROJECT_STATE.json`。

### 📊 1. 市场调研

**必须回答的问题：**
- 目标市场是什么？规模多大？
- 市场增长趋势如何？
- 主要的市场参与者有哪些？
- 市场进入壁垒是什么？

**搜索建议：**
- 使用 Web Search 搜索行业报告
- 查找市场分析文章
- 搜索投融资新闻

**输出格式：**
```markdown
## 市场调研结果

### 目标市场
[描述]

### 市场规模
[数据]

### 增长趋势
[分析]

### 市场参与者
- [参与者1]
- [参与者2]

### 进入壁垒
[分析]
```

完成后询问：**市场调研已完成，确认保存？(y/n)**

---

### ⚙️ 2. 技术研究

**必须回答的问题：**
- 实现这个产品需要哪些核心技术？
- 有哪些现成的技术方案/开源项目可用？
- 技术难点和风险是什么？
- 团队是否具备所需技术能力？

**搜索建议：**
- GitHub 搜索相关开源项目
- 查找技术博客和最佳实践
- 查阅官方文档

**输出格式：**
```markdown
## 技术研究结果

### 核心技术需求
- [技术1]: [用途]
- [技术2]: [用途]

### 可用方案对比
| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| [方案A] | | | |
| [方案B] | | | |

### 技术难点
1. [难点1] - 应对策略：[策略]
2. [难点2] - 应对策略：[策略]

### 推荐技术栈
- 前端: [选择]
- 后端: [选择]
- 数据库: [选择]
- 部署: [选择]
```

### 👥 3. 用户研究

**必须回答的问题：**
- 目标用户是谁？用户画像是什么？
- 用户的核心痛点是什么？
- 用户目前如何解决这些问题？
- 用户愿意为解决方案付费吗？

**输出格式：**
```markdown
## 用户研究结果

### 用户画像
- 年龄段:
- 职业:
- 技术水平:
- 使用场景:

### 用户痛点
1. [痛点1] - 严重程度：高/中/低
2. [痛点2] - 严重程度：高/中/低

### 现有解决方案
- [方案1]: 用户评价 [评价]
- [方案2]: 用户评价 [评价]

### 付费意愿
[分析]
```

### ⚖️ 4. 合规研究

**必须回答的问题：**
- 有哪些相关法规需要遵守？
- 数据隐私要求是什么（GDPR/个人信息保护法等）？
- 行业特定的合规要求？
- 需要哪些资质或认证？

**输出格式：**
```markdown
## 合规研究结果

### 适用法规
- [法规1]: [影响]
- [法规2]: [影响]

### 数据隐私要求
- 数据收集: [要求]
- 数据存储: [要求]
- 用户权利: [要求]

### 行业合规
[具体要求]

### 所需资质
- [ ] [资质1]
- [ ] [资质2]
```

### 🎯 6. 竞品研究

## 研究完成检查

完成所有 6 类研究后：

1. 生成 `docs/RESEARCH.md` 汇总所有研究结果
2. 更新 `PROJECT_STATE.json`:
```json
{
  "phases": {
    "research": { "status": "completed", "completedAt": "[timestamp]" },
    "planning": { "status": "in_progress", "completedAt": null }
  },
  "currentPhase": 2
}
```
